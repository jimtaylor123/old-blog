How I built this blog

 January 1, 2020 | Image: Kevin Jarrett

As a way of giving credit to others AND of blowing my own trumpet, here's a quick overview of the tools I used to make this blog:

VS Code

VS code is the most popular IDE (aka code editing software) in the world. And is developed free of charge by Microsoft. I don't know WHY it's free, I don't think anyone knows, but I'm guessing it's a shrewd move by Microsoft to get the attention of the developer led spend, and along with their purchase of github it has clearly changed the image of the company massively.

Why not PHP storm? I often work with developers who prefer PHP storm, but I like the fact that VS code has such a large and diverse user base, meaning it's not commited to a particular language. As someone who aspires to be a full stack developer and a technological generalist/pragmatist that's why I'm more attracted to VS code that any other editor.

And choosing your code editor is a bit like an investment - once you've chosen it you stay with it unless things get really bad.

Finally, VS code is really great, and it's free.

Static site generator: Laravel Jigsaw

I've been developing with Laravel for 4 years or so. When I decided to blog I tried various platforms such as Laravel wink and wordpress before settling on Jigsaw.

I'm not going to lie, Jigsaw was more complicated than I expected to set up, but truth be told it's a great static site generator and I can't imagine it could be made any more user friendly without sacrificing it's aim to be a site generator for developers. I highly recommend it.

Design: Tailwind Toolbox

Tailwind toolbox is where I got my homepage design from, specifically this profile page. The whole site is full of great starters, and I aim to return to the site in future for other projects.

I also like the fact that the project originates in India - sometimes the Laravel community feels very US based, and I think developing world agencies/programmers are generally underrated - it's just good to keep your influences broad and varied.

CSS: Tailwind CSS

I absolutely love tailwind CSS - the best thing about it is that you learn CSS by using the utility classes. In other words, if you are struggling to expand the amount of CSS you understand OR use, start using tailwind and your style and capabilities will suddently expand.

A quick example - I would never have thought of using hairline text or antialized fonts before using tailwind. Same goes for things like letter height ("leading", as in "lead" roofing) and tracking (letter spacing):

<body class="font-sans antialiased text-gray-100 leading-normal tracking-wider p-0 bg-cover" style="background-image:url('/assets/images/night.jpg');">

Another example - the flex utilities in tailwind are a summary of the possibilities offered by flexbox. Using tailwind makes you understand the range of options as you code.

Images and icons

No surprises here:

  • I'm constantly amazed by the free images on Unsplash images and grateful to be able to use them with credits on my blog. For my day image I have used a photo by Cassiano Psomas of a beach in Florianopolis that I loved (both the beach and the image). And for my night image I used an evocative image of London by Tim Arterbury.
  • SVG icons from Simple icons and Font Awesome were all helpful, as usual.

I was planning to use zondicons, but found the range too limited - they are excellent and free so I can't complain.

Disqus comments

I just used the standard disqus code embed. I considered and rejected both twitter based comments and hosting my own. Honestly, I don't forsee comments being a big deal, it's just a personal blog.

Github pages

Finally, I'm using github pages to host my blog completely free. Another Microsoft freebie.
